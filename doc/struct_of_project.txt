проект состоит из трех библиотек 
    libraries/application
    libraries/functors
    libraries/repository
и главного приложения
    sources

aplication - отвечает за каркас приложения, 
управление главным циклом исполнения команд исполнения
и обработку ошибок.
functors - содержит код команд. отвечает за логику отдельных команд.
repository - отвечает за загрузку\выгрузку и хранение изображений.
sources - содержит код запуска и настройки приложения. 

зависимости:
opencv last version.
catch2 - unit tests framework (опционально, по умолчанию выключено)


сборка и запуск приложения:

git clone ....
cd improc
mkdir build
cd build
cmake ..
make -j8
./sources/improc

если в корневом файле CMakeLists.txt
перевести соответствующие флаги в состояние ON,
то первый флаг заставит cmake при каждой сборке проекта собирать юнит тесты.
А второй флаг заставит ещё и запускать их после каждой сборке проекта автоматически (очень удобно).
option(BUILD_UNIT_TESTING "Enable unit tests building" OFF)
option(RUN_UNIT_TESTS "Enable unit tests running after building." OFF)
сами файлы сетов лежат всегда рядом с тестируемым кодом и имеют расширение *.cxx
cmake сам обходит все библиотеки и собирает тесты в одну тестовою сборку run_unit_tests.


Я обычно дома работаю в Linux. Соответственно все проверил под этой ОС.
Но код кросплатформенный все должно собираться и работать на любой платформе.
